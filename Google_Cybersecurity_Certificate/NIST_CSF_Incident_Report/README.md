# NIST CSF Incident Report: DoS Attack Analysis

## Overview

This project applies the National Institute of Standards and Technology 
Cybersecurity Framework (NIST CSF) to analyse a real-world DoS attack 
scenario. It was completed as a portfolio activity for the Google 
Cybersecurity Certificate (Course 3: Connect and Protect: Networks and 
Network Security).

## Scenario

A multimedia company offering web design, graphic design, and social media 
marketing services experienced a Denial of Service (DoS) attack that 
disrupted internal network operations for approximately two hours. A malicious 
actor flooded the network with ICMP packets through an unconfigured firewall, 
making all internal resources inaccessible to staff.

## What I Did

Applied all five NIST CSF core functions to the incident:

- **Identify** - Determined the attack type (ICMP flood DoS), identified 
affected systems, and identified the root cause (unconfigured firewall)
- **Protect** - Recommended firewall rules, source IP verification, regular 
audits, and staff training
- **Detect** - Proposed SIEM deployment, IDS/IPS implementation, and network 
baseline monitoring
- **Respond** - Outlined containment steps, service management, investigation 
process, and neutralization procedures
- **Recover** - Documented service restoration using backups, incident 
documentation, and stakeholder communication plan

## Skills Demonstrated

- NIST Cybersecurity Framework application
- Incident analysis and documentation
- Network security concepts (DoS, ICMP flood, firewall configuration)
- Security hardening recommendations
- Stakeholder communication
